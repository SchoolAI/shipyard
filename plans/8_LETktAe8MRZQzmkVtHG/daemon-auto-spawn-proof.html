Daemon Auto-Spawn on MCP Startup - Proof

Test Flow

1. Kill any existing daemon
2. Start MCP server
3. Daemon auto-spawns (detached)
4. Kill MCP server  
5. Daemon survives

Self-Propagating Bootstrap

MCP checks: Is daemon running?
  → fetch('http://localhost:56609/health')
  → NO (404/timeout)

MCP spawns detached daemon:
  → spawn('node', ['daemon/dist/index.js'], {
      detached: true,
      stdio: 'ignore'
    })
  → daemon.unref()

Daemon starts on port 56609

MCP exits
Daemon survives (detached process)

Implementation

File: apps/server/src/daemon-launcher.ts
Function: ensureDaemonRunning()

Called from: apps/server/src/index.ts
Location: After registry initialization, before MCP server starts

Ports: [56609, 49548] with fallback
Timeout: 5 seconds for daemon to become ready

Verification

✓ MCP started (PID: 73996)
✓ Daemon running
✓ Daemon survived MCP exit
